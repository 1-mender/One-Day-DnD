Основные проблемы и риски (приоритеты)

Высокий:
Инвентарь: race conditions при трансферах/редактировании — потеря/дубли предметов.
Авторизация/имперсонация: риск эскалации прав при ошибках setup/auth.
Средний:
Загрузка больших файлов/изображений при импорте Bestiary — лимиты и таймауты.
Матчмейкинг: тайминги и нагрузка при пиковых очередях.
Низкий:
Доступность (a11y) мелких компонентов и меню (например, BottomNav).
Конкретные улучшения (приоритет + действия)

Ключевые направления

Аудит: пройти client/src/components и сопоставить экраны → выявить inconsistent patterns. (начато)
Доступность: убедиться, что BottomNav, Modal, ActionMenu корректно работают с клавиатурой и aria‑атрибутами.
Производительность: ленивый загрузчик изображений, виртуализация длинных списков (bestiary, каталоги), код‑сплиттинг.
Мобильность: улучшить touch‑мишени, swipe/drag для инвентаря, адаптивные лэйоуты.
UX flows: явные загрузки/ошибки/пустые состояния; прогресс при больших импорт/экспорт задачах.
Стабильность данных: показывать состояние ожидания/подтверждения при трансферах, откаты и понятные ошибки при конфликтах.
Инструментация: собрать UX‑метрики (time-to-interactive, first-input-delay), логировать пользовательские ошибки и fail‑cases.
Design System: ввести токены (цвета, размеры, отступы), общий набор компонентов и документацию.
Конкретные рекомендации (быстро выполнить)

Loading States: добавить skeletons и spinners для всех списков (bestiary, inventory) вместо пустого контента.
Optimized Images: применять srcset/WebP + lazy loading; на сервере поддержать thumbnails для списка монстров.
Virtualization: для long lists использовать windowing (react-window / react-virtuoso).
Debounce / Optimistic UI: при изменениях инвентаря применить optimistic updates + rollback при ошибке; дебаунс для поиска/фильтров.
Error UX: централизовать отображение ошибок (чёткие CTA — retry/cancel), добавить user-friendly сообщения из api.js.
Forms & Validation: inline validation + focus на первом ошибочном поле; prevent double-submit.
Touch & Gestures: увеличить target size до 44–48px для кнопок, добавить swipe-to-open/close для меню на мобилке.
Accessibility: проверить tab order, aria roles for menus/modals, visible focus styles, screen reader labels.
Onboarding / First-run: быстрый guide или hint для DM setup и join flow (join code).
Progress for background tasks: для импорта/экспорта добавить job queue + pollable status endpoint и прогрессбар.
Analytics: события для ключевых флоу: transfer.created/accepted/rejected, bestiary.import.completed, matchmaking.queue.enter.
Microcopy: ясные labels и подтверждения при критичных действиях (удаление, transfer).
Low-effort, high-impact

Клиентская проверка размера файла перед отправкой.
Показ friendly ошибки при сетевых сбоях и авто‑повтор при временных ошибках.
Добавить keyboard shortcut для основных действий (например, открыть BottomNav меню).
