## ОПТИМАЛЬНАЯ КОНФИГУРАЦИЯ

### Production конфигурация

#### `.env` для production

```bash
# Сервер
PORT=3000
NODE_ENV=production
LOG_LEVEL=info

# Безопасность
CSP_DISABLED=0
CSP_ALLOW_UNSAFE_EVAL=0
CSP_UPGRADE_INSECURE_REQUESTS=0

# Данные
DND_LAN_DATA_DIR=/var/lib/dnd-lan
BACKUP_DIR=/var/lib/dnd-lan/backups

# Производительность
NODE_OPTIONS=--max_old_space_size=1024
```

#### Nginx reverse proxy (опционально)

```nginx
upstream dnd_app {
  server 127.0.0.1:3000;
}

server {
  listen 80;
  server_name dnd.local;
  
  # Redirects to https (если используешь SSL)
  # return 301 https://$server_name$request_uri;
  
  location / {
    proxy_pass http://dnd_app;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    
    # WebSocket
    proxy_read_timeout 86400;
  }
  
  # Static files caching
  location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
  }
}
```

#### Systemd сервис (Linux)

```ini
# /etc/systemd/system/dnd-lan.service

[Unit]
Description=D&D LAN Server
After=network.target

[Service]
Type=simple
User=dndlan
WorkingDirectory=/opt/dnd-lan
ExecStart=/usr/bin/npm run start
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

Использование:
```bash
sudo systemctl enable dnd-lan
sudo systemctl start dnd-lan
sudo systemctl status dnd-lan
sudo journalctl -u dnd-lan -f
```

#### Docker конфигурация (опционально)

```dockerfile
# Dockerfile
FROM node:20-alpine

WORKDIR /app

# Установить зависимости
COPY package*.json ./
RUN npm install --production

# Копировать код
COPY . .

# Build клиента
WORKDIR /app/client
RUN npm install && npm run build

# Back to root
WORKDIR /app

# Expose порт
EXPOSE 3000

# Запуск
CMD ["npm", "run", "start"]
```

```yaml
# docker-compose.yml
version: '3.8'

services:
  dnd-lan:
    build: .
    ports:
      - "3000:3000"
    volumes:
      - dnd_data:/app/server/data
    environment:
      - NODE_ENV=production
      - LOG_LEVEL=info
    restart: unless-stopped

volumes:
  dnd_data:
    driver: local
```

### Мониторинг Health endpoints

```bash

# С jq для красивого вывода
watch -n 10 'curl -s http://localhost:3000/readyz | jq .'

# Скрипт мониторинга
#!/bin/bash
while true; do
  status=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/readyz)

  else
    echo "✅ Server healthy"

  sleep 10
done
```

## БЕЗОПАСНОСТЬ

### CSRF защита

```javascript
// Включена по умолчанию
// Все POST/PUT/DELETE требуют CSRF токен

// В HTTP заголовках
X-CSRF-Token: <token>

// Или в cookie
csrf_token=<value>
```

### CORS (Cross-Origin Resource Sharing)

```javascript
// Development (DEV_ALLOWED_ORIGINS)
DEV_ALLOWED_ORIGINS=http://localhost:5173,http://localhost:3001

// Production
// CORS отключен (только same-origin)
```

### Content Security Policy (CSP)

```javascript
// Включена по умолчанию
// Запрещает inline скрипты, eval и т.д.

// Если нужна unsafe-eval (development)
CSP_ALLOW_UNSAFE_EVAL=1

// Отключить CSP (не рекомендуется)
CSP_DISABLED=1
```

### Encryption чувствительных данных

```javascript
// Пароли - bcryptjs хеширование
const hash = await bcrypt.hash(password, 10);

// JWT токены
const token = jwt.sign({ userId }, secret, { expiresIn: '24h' });

// Cookies - secure, httpOnly
secure: true          // Только HTTPS
httpOnly: true        // Недоступен JavaScript
sameSite: 'lax'       // CSRF защита
```

### Rate limiting

```javascript
// Включен на API маршрутах
// Обрабатывает: POST /api/auth, PUT /api/inventory

// Лимиты (configurable)
- 100 requests / 15 minutes (auth)
- 1000 requests / 15 minutes (general)
```

### SQL injection защита

```javascript
// Используются prepared statements (параметризованные запросы)
// БЕЗОПАСНО:
db.prepare('SELECT * FROM users WHERE id = ?').get(userId);

// НЕБЕЗОПАСНО (НЕ ИСПОЛЬЗУЙ):
db.prepare(`SELECT * FROM users WHERE id = ${userId}`).get();
```

### Input validation

```javascript
// Все входные данные валидируются
// - JSON parsing
// - Type checking
// - Constraints проверка
// - Sanitization

// Пример валидации

  username: { type: 'string', min: 3, max: 32 },
  password: { type: 'string', min: 8 },
  email: { type: 'email' }
```

## УСТРАНЕНИЕ ПРОБЛЕМ

### Проблема: Сервер не запускается

**Признаки:**
```
Error: EADDRINUSE: address already in use :::3000
Error: Cannot find module
Cannot connect to database
```

**Решения:**
```bash
# 1. Проверить занятый port
netstat -ano | findstr :3000         # Windows
lsof -i :3000                        # Linux/macOS

# 2. Убить процесс
taskkill /PID <PID> /F               # Windows
kill -9 <PID>                        # Linux/macOS

# 3. Переустановить зависимости
rm -rf node_modules server/node_modules client/node_modules
npm install && npm --prefix server install && npm --prefix client install

# 4. Проверить файлы
npm run check:utf8                   # Проверить кодировку

# 5. Запустить с debug логами
LOG_LEVEL=debug npm run dev
```

### Проблема: Игроки не видят друг друга

**Решения:**
```bash
# 1. Проверить WebSocket соединение
# DevTools → Network → WS → socket.io

# 2. Проверить firewall
# Убедиться порт 3000 открыт между машинами

# 3. Проверить IP адреса
ipconfig                             # Windows
ip addr                              # Linux

# 4. Проверить CORS
DEV_ALLOWED_ORIGINS=* npm run dev

# 5. Проверить readyz
curl http://localhost:3000/readyz
```

### Проблема: БД заблокирована

**Решения:**
```bash
# 1. Ждать 30 секунд (обычно разблокируется)
# 2. Перезагрузить сервер
# 3. Проверить другие процессы
lsof server/data/app.db              # Linux/macOS

# 4. Если нужно сбросить БД
rm server/data/app.db
# Сервер создаст новую БД при старте
```

### Проблема: Медленная загрузка

**Решения:**
```bash
# 1. Проверить производительность
npm run perf:budget

# 2. Оптимизировать изображения
npm run images:opt

# 3. Увеличить памяти Node
NODE_OPTIONS="--max_old_space_size=2048" npm run dev

# 4. Проверить сеть
curl -w "%{time_total}\n" -o /dev/null -s http://localhost:3000/readyz
```

### Проблема: WebSocket переподключения

**Решения:**
```bash
# 1. Проверить логи сервера
LOG_LEVEL=debug npm run dev

# 2. Проверить стабильность сети
ping -t <server-ip>                  # Windows
ping -c 5 <server-ip>               # Linux/macOS

# 3. Увеличить timeout
# client/src/socket.js
reconnectionDelay: 2000,
reconnectionAttempts: 20,

# 4. Проверить firewall и NAT
```

### Проблема: "Insufficient disk space"

**Решения:**
```bash
# 1. Проверить место
df -h                                # Linux/macOS
diskutil info /                      # macOS

# 2. Очистить старые backup'ы
rm server/data/backups/backup-*.zip  # Оставить последние 5

# 3. Очистить uploads
npm --prefix server run cleanup:uploads

# 4. Запустить preflight
npm run preflight
```

## DEVOPS И CI/CD

### GitHub Actions пример

```yaml
# .github/workflows/verify.yml
name: Verify

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v2
      - uses: actions/setup-node@v2
        with:
          node-version: '20'
      
      - name: Install dependencies
        run: |
          npm install
          npm --prefix server install
          npm --prefix client install
      
      - name: Run verification
        run: npm run verify
      
      - name: Upload artifacts
        if: always()
        uses: actions/upload-artifact@v2
        with:
          name: test-results
          path: artifacts/
```

### Deploy скрипт

```bash
#!/bin/bash
# deploy.sh

set -e

echo "🚀 Deploying DnD LAN..."

# 1. Pull latest code
git pull origin main

# 2. Install dependencies
npm install
npm --prefix server install
npm --prefix client install

# 3. Run verification
npm run verify

# 4. Build
npm run build

# 5. Create backup
curl -X POST http://localhost:3000/api/backup/create

# 6. Stop service
sudo systemctl stop dnd-lan || true

# 7. Copy files
sudo cp -r . /opt/dnd-lan/

# 8. Start service
sudo systemctl start dnd-lan

# 9. Check health
sleep 2
curl http://localhost:3000/readyz

echo "✅ Deploy complete!"
```

### Мониторинг и alerting

```bash
#!/bin/bash
# monitor.sh - Простой мониторинг

while true; do
  status=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/readyz)
  
  if [ "$status" != "200" ]; then
    # Отправить алерт
    echo "Server down! Status: $status" | mail -s "Alert" admin@example.com
    
    # Попытаться перезагрузить
    sudo systemctl restart dnd-lan
    sleep 5
  fi
  
  sleep 60
done
```
