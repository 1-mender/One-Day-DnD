## 🚀 БЫСТРЫЙ СТАРТ (5 минут)

### Шаг 1: Установка

```bash
cd "dnd-lan"
npm install
npm --prefix server install
npm --prefix client install
```

### Шаг 2: Запуск

```bash
npm run dev
```

### Шаг 3: Открыть в браузере

```
Мастер (DM):  http://localhost:3000/dm
              или http://localhost:5173/dm (Vite dev)

Игроки:       Сканируют QR из DM Dashboard
```

**Готово! 🎲**

## 🔧 ЧАСТЫЕ НАСТРОЙКИ

### Изменить порт

```bash
# В PowerShell / Terminal
$env:PORT = 3001
npm run dev

# или в .env файле
PORT=3001
```

### Включить debug логирование

```bash
LOG_LEVEL=debug npm run dev
```

### Путь к данным БД

```bash
# Windows (автоматически)
%LOCALAPPDATA%\dnd-lan\app.db

# Linux/macOS (или явно)
DND_LAN_DATA_DIR=/path/to/data npm run dev
```

### CORS для развития

```bash
# Если клиент на другом host
DEV_ALLOWED_ORIGINS=http://192.168.1.5:5173 npm run dev
```

## 🔐 БЕЗОПАСНОСТЬ И БЭКАПЫ

### Резервная копия вручную

```bash
# В интерфейсе DM Dashboard
→ Settings → Create Backup

# Или через API
curl -X POST http://localhost:3000/api/backup/create
```

### Восстановление из резервной копии

```bash
# В интерфейсе
→ Settings → Restore Backup

# Выбрать файл .zip
```

### Автоматические бэкапы

- Создаются каждые 30 минут (при активности)
- Хранятся в `server/data/backups/`
- Старые удаляются автоматически (>50 файлов)

## 🌐 ПОДКЛЮЧЕНИЕ ИЗ СЕТИ

### Найти IP адрес машины DM

```bash
# Windows (PowerShell)
ipconfig

# Linux/macOS
ifconfig
# или
hostname -I
```

### Подключить игроков

1. Игроки в одной WiFi сети
2. Открыть `http://<DM-IP>:3000/player`
3. Вводят код присоединения (из DM Dashboard)
4. DM подтверждает
5. Игрок может играть

**Пример:** Если IP=192.168.1.100:
```
http://192.168.1.100:3000/player
```

## 🏗️ PRODUCTION DEPLOY

### Собрать для production

```bash
npm run build
```

Результат: `dnd-lan/server/public/` готов

### Запустить production версию

```bash
npm run start

# или явно
NODE_ENV=production npm run start
```

**Результат:**
- Один сервер на порту 3000
- Обслуживает собранный клиент
- Оптимизированный код
- Минифицирован

### Checklist перед production

```bash
# 1. Проверить security
npm run audit:server
npm run audit:client

# 2. Проверить производительность
npm run perf:budget

# 3. Полная верификация
npm run verify

# 4. Preflight проверка
npm run preflight

# 5. Создать базовый backup
npm run dev
# В DM Dashboard → Create Backup
```

## 📦 СТРУКТУРА ФАЙЛОВ ПРИ СТАРТЕ

### После `npm install`:

```
dnd-lan/
├── node_modules/           # Корневые зависимости
├── client/
│   ├── node_modules/       # Клиентские зависимости
│   ├── dist/               # Собранный клиент (после build)
│   └── ... (исходники)
├── server/
│   ├── node_modules/       # Серверные зависимости
│   ├── data/
│   │   └── app.db         # БД (создается при первом запуске)
│   ├── public/            # Собранный фронтенд (после build)
│   └── ... (исходники)
└── shared/
    └── node_modules/      # Общие зависимости
```

### После `npm run dev`:

Добавляются:
```
dnd-lan/
├── server/
│   ├── data/
│   │   ├── app.db         # БД создана
│   │   ├── backups/       # Папка для резервных копий
│   │   └── tmp/           # Временные файлы
│   └── uploads/           # Загруженные файлы
└── client/
    └── dist/              # Собран только для production сборки
```

## 🐛 РЕШЕНИЕ ЧАСТЫХ ПРОБЛЕМ

### "Port 3000 already in use"

```bash
# Windows (PowerShell)
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# Linux/macOS
lsof -i :3000
kill -9 <PID>

# Или просто используй другой port
PORT=3001 npm run dev
```

### "Cannot find module..."

```bash
# Переустановить зависимости
rm -rf node_modules server/node_modules client/node_modules
npm install
npm --prefix server install
npm --prefix client install
```

### "Database locked"

```bash
# Обычно проходит сам за 30 сек
# Если нет - перезагрузить сервер:
npm run dev
# (Ctrl+C и запустить заново)
```

### "WebSocket connection failed"

```bash
# Проверить что порт открыт
curl http://localhost:3000/readyz

# Проверить CORS (dev only)
DEV_ALLOWED_ORIGINS=* npm run dev

# Проверить firewall
# Убедиться порт 3000 не заблокирован
```

### "Game crashes при загрузке карты"

```bash
# Проверить браузер консоль (DevTools → Console)
# Проверить серверные логи (npm run dev)

# Может помочь:
# 1. F5 refresh
# 2. Ctrl+Shift+Delete (очистить cache)
# 3. Перезагрузить сервер (Ctrl+C, npm run dev)
```
