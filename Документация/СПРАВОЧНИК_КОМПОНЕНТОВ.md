### Context API (Global State)

#### `AuthContext.jsx`

```javascript
// Содержит:
{
  user: { id, username, role }      // Текущий пользователь
  isAuthenticated: bool              // Статус входа
  login(username, password)          // Функция входа
  logout()                           // Функция выхода

// Использование:
const { user, isAuthenticated, login } = useContext(AuthContext);
```

```javascript
// Содержит:
```

```javascript
function createApp() {
  const app = express();
  
  // 1. Security middleware
  app.use(helmet({ ... }));
  app.use(csrf-protection);
  
  // 2. Logging
  app.use(httpLogger);

    app.use(cors-handler);

  app.use('/minigames', express.static(minigamesDir));

  app.use('/api/auth', authRouter);
  app.use('/api/party', partyRouter);

  // ... другие маршруты
  
  // 7. Health endpoints

  // 8. SPA fallback
  app.get('*', (req, res) => {
    res.sendFile(path.join(publicDir, 'index.html'));
  });
  
  return app;
}
```

```javascript
export const inventoryModule = {

      .prepare('SELECT * FROM inventory WHERE character_id = ?')
      .all(characterId);

  // Добавить предмет
  addItem(characterId, itemId, quantity) {
    const existing = getDb()
      .prepare('SELECT * FROM inventory WHERE character_id = ? AND item_id = ?')
      .get(characterId, itemId);
    
    if (existing) {

      getDb()
```

## СТРУКТУРА БД

### Таблицы SQL

```sql
-- Parties
CREATE TABLE parties (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  description TEXT,
  created_at TEXT,
  updated_at TEXT
);

-- Players
CREATE TABLE players (
  id TEXT PRIMARY KEY,
  party_id TEXT,
  name TEXT NOT NULL,
  character_name TEXT,
  status TEXT DEFAULT 'connected',
  joined_at TEXT,
  FOREIGN KEY (party_id) REFERENCES parties(id)
);

-- Characters
CREATE TABLE characters (
  id TEXT PRIMARY KEY,
  player_id TEXT,
  name TEXT NOT NULL,
  class TEXT,
  race TEXT,
  level INTEGER DEFAULT 1,
  hp INTEGER,
  max_hp INTEGER,
  ac INTEGER,
  FOREIGN KEY (player_id) REFERENCES players(id)
);

-- Inventory
CREATE TABLE inventory (
  id TEXT PRIMARY KEY,
  character_id TEXT,
  item_id TEXT,
  quantity INTEGER,
  slot INTEGER,
  FOREIGN KEY (character_id) REFERENCES characters(id)
);

-- Bestiary
CREATE TABLE bestiary (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  type TEXT,
  hp INTEGER,
  ac INTEGER,
  challenge_rating INTEGER,
  description TEXT
);

-- Maps
CREATE TABLE maps (
  id TEXT PRIMARY KEY,
  party_id TEXT,
  name TEXT,
  image_data BLOB,
  created_at TEXT,
  FOREIGN KEY (party_id) REFERENCES parties(id)
);

-- Backups
CREATE TABLE backup_history (
  id TEXT PRIMARY KEY,
  party_id TEXT,
  file_path TEXT,
  created_at TEXT,
  size_bytes INTEGER,
  FOREIGN KEY (party_id) REFERENCES parties(id)
);

-- Sessions
CREATE TABLE sessions (
  id TEXT PRIMARY KEY,
  user_id TEXT,
  token TEXT,
  expires_at TEXT,
  created_at TEXT
);

-- Events
CREATE TABLE events (
  id TEXT PRIMARY KEY,
  party_id TEXT,
  type TEXT,
  data JSON,
  created_at TEXT,
  FOREIGN KEY (party_id) REFERENCES parties(id)
);
```

### Связи между таблицами

```
parties (1) ──────────────(N) players
            ├──────────────────(N) characters
            ├──────────────────(N) maps
            └──────────────────(N) events

players (1) ───────────────(N) characters
                           ├──(N) inventory
                           └──(N) sessions

bestiary (independent catalog)
backup_history (references parties)
```

## ИНТЕГРАЦИЯ МОДУЛЕЙ

### Инициализация при старте

```
1. index.js запускается
2. runStartup() - проверка конфигурации
3. initDb() - инициализация БД (schema.sql + миграции)
4. createApp() - создание Express
5. createSocketServer() - Socket.io
6. setupHealth() - health endpoints
7. startBackgroundJobs() - фоновые задачи
8. registerShutdown() - обработчики выключения
9. httpServer.listen() - слушаем порт
```

### Поток данных при присоединении игрока

```
1. Игрок открывает URL: http://localhost:3000/player
2. Браузер запрашивает /player (Express возвращает index.html)
3. React загружается (App.jsx)
4. Socket.io клиент подключается к серверу
5. Socket.io сервер создает новое соединение
6. Клиент отправляет 'player:join' событие
7. Сервер создает запись в таблице players
8. Broadcast 'player:joined' для всех клиентов
9. DM Dashboard обновляется (Socket.on())
10. DM подтверждает присоединение
11. Игроку отправляются данные party и персонажа
12. Игрок видит свой инвентарь и персонажа
```

### Цикл обновления инвентаря

```
1. Игрок нажимает "Поднять предмет"
2. Клиент: api.post('/api/inventory/add', { itemId, qty })
3. Сервер: inventoryRouter обрабатывает POST
4. inventoryModule.addItem() обновляет БД
5. Сервер: io.emit('inventory:changed', { ... })
6. Все клиенты получают событие
7. InventoryContext обновляется
8. React re-render Inventory компонент
9. Игрок видит новый предмет
```

### Система резервного копирования

```
1. DM нажимает "Create Backup" на Settings
2. Клиент: POST /api/backup/create
3. Сервер: backup.js создает ZIP всей БД
4. Сохраняет в server/data/backups/
5. Возвращает backup_id
6. DM видит в списке backup'ов
7. Для восстановления: POST /api/backup/:id/restore
8. Сервер: распаковывает ZIP → восстанавливает БД
9. Все игроки переподключаются автоматически
```
